# Checklist: Calidad de Código Generado por IA

## Propósito
Validar calidad de código generado o asistido por IA antes de aceptar.

## Checklist

### Estructura y Organización
- [ ] Código bien organizado y estructurado
- [ ] Sigue patrones del proyecto
- [ ] Sin código duplicado innecesario
- [ ] Separación de responsabilidades clara

### Legibilidad
- [ ] Nombres descriptivos
- [ ] Código fácil de entender
- [ ] Complejidad ciclomática razonable
- [ ] Sin "magic numbers"

### Documentación
- [ ] Docstrings/comentarios presentes
- [ ] Documentación clara y útil
- [ ] Type hints incluidos (si aplica)

### Estándares
- [ ] Sigue PEP 8 / estándares del proyecto
- [ ] Formatter aplicado
- [ ] Linter pasa sin errores críticos

### Funcionalidad
- [ ] Lógica correcta y completa
- [ ] Casos edge manejados
- [ ] Validación de inputs adecuada
- [ ] Manejo de errores robusto

### Tests
- [ ] Tests presentes
- [ ] Cobertura adecuada (>80%)
- [ ] Tests son mantenibles
